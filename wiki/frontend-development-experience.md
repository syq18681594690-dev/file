# 前端开发经验

## 1. 适用范围

本文档用于持续记录前端开发中的技术选型、实现约定、问题处理经验和验证方法。

当前仅确认需要建立前端开发经验页面，尚未指定框架、语言版本、构建工具或工程目录结构。因此，本页面先作为后续经验沉淀入口，不将未确认内容写成强制规范。

## 2. 经验记录分类

后续新增前端经验时，按实际内容归入以下分类：

| 分类 | 记录内容 |
| --- | --- |
| 技术栈与依赖 | 框架、语言、包管理器、UI 组件库及版本约束 |
| 项目结构 | 目录划分、组件组织、路由、状态管理与请求层组织方式 |
| 页面与组件 | 组件复用边界、表单、列表、弹窗及交互处理经验 |
| 样式与体验 | 布局、响应式、主题、可访问性与交互反馈约定 |
| 接口联调 | 请求封装、错误处理、鉴权、字段适配与前后端协作规则 |
| 质量与交付 | 测试、构建、部署、性能优化及问题排查记录 |

## 3. 记录原则

- 优先记录已实际采用或已验证有效的实践，并说明适用场景。
- 对技术选型明确记录版本、选择原因和替代方案；尚未确定的选项列入待确认事项。
- 对问题处理经验记录现象、原因、解决方式和验证结果，避免只保留结论。
- 涉及接口约定时，必要时与 [后端开发约定](backend-development-guide.md) 建立交叉引用。
- 同一功能在不同页面使用时，遵循 [跨页面功能一致性约定](cross-page-feature-consistency.md)，优先统一复用并在存在特殊差异时告知用户。

## 4. 待后续补充

- 前端框架及版本，例如 Vue、React 或其他方案。
- 使用 TypeScript 或 JavaScript 的约定。
- 包管理器、构建工具与启动命令。
- UI 组件库、CSS 方案及页面风格规范。
- API 请求、鉴权、错误提示与状态管理方式。
- 测试、代码检查、构建发布及性能验收方式。
