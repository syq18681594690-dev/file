# 跨页面功能一致性约定

## 1. 适用范围

本约定适用于前端页面和支撑这些页面的后端接口、业务逻辑。当同一业务功能需要在不同页面或不同入口中使用时，前后端均应遵循本约定。

## 2. 核心要求

- 同一个功能被多个页面引用时，应保持功能行为、业务规则、数据口径和交互含义一致。
- 不应为不同页面分别实现多套本质相同但细节不一致的方案。
- 前端应优先复用统一组件、组合式逻辑、请求封装或状态处理规则，具体复用形态依据后续技术栈确定。
- 后端应优先复用统一接口契约、服务逻辑、校验规则和返回口径，避免页面差异渗透成互相矛盾的业务实现。

## 3. 允许差异的条件

- 页面确实存在不同业务场景、权限限制、展示目的或交互要求时，可以在统一基础能力上增加明确差异。
- 实施特殊处理前，应向用户说明差异原因、受影响页面以及与通用方案不同的行为。
- 经用户确认的差异，应记录在对应前端或后端规范页面中；若影响多个页面，应同时回写本页面或建立关联说明。

## 4. 实施检查

- 新增页面复用已有功能前，先确认是否已有对应组件、接口或业务规则可复用。
- 修改共享功能时，检查所有引用页面和关联接口是否仍保持一致。
- 如发现已有多个实现存在冲突，先向用户说明现状与统一方案，再决定是否调整既有行为。
